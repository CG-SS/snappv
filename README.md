# Snappv

A pure [V](https://vlang.io/) implementation of the [Snappy](https://github.com/google/snappy) compression format.

## Features

- **Raw block format** — Compress and decompress single, unframed Snappy blocks.
- **Framing format** — Full support for the Snappy stream framing format (as defined in the Snappy specification), enabling chunked compression/decompression of large payloads.
- **Pure V** — No C dependencies; the entire implementation is written in V.
- **Static CRC32C table** — Precomputed lookup table for fast CRC32C checksums, generated via `./scripts/generate_crc32c_table.py`.

## Usage

### Raw Block Compression

```v
import snappv

fn main() {
    data := 'hello snappy! '.repeat(100)

    // Compress
    compressed := snappv.compress(data.bytes())

    // Decompress
    decompressed := snappv.decompress(compressed) or { panic(err) }

    assert decompressed == data.bytes()
}
```

### Framing Format (Streaming)

If you already have the full stream bytes, you can use the one-shot functions:

```v
import snappv

fn main() {
    data := 'hello snappy! '.repeat(10000)

    // Compress to framed stream
    framed := snappv.encode_stream(data.bytes())

    // Decompress from framed stream
    decompressed := snappv.decode_stream(framed) or { panic(err) }

    assert decompressed == data.bytes()
}
```

Otherwise, you can use StreamEncoder and StreamDecoder to do it in batches:

```v
import snappv

fn main() {
	original_text := 'Hello, Snappy framing format! ' +
	'This is a longer string to demonstrate chunking behavior ' +
	'since the max chunk size is 64 KiB. We could repeat this ' +
	'to fill multiple chunks if needed.'

	original_bytes := original_text.bytes()

	// --- Encoding ---
	mut encoder := snappv.StreamEncoder{}

	// Write the data in chunks (simulating incremental writes)
	chunk_size := 20
	mut pos := 0
	for pos < original_bytes.len {
		end := if pos + chunk_size < original_bytes.len { pos + chunk_size } else { original_bytes.len }
		encoder.write(original_bytes[pos..end]) or { panic('encode write error: ${err}') }
		pos = end
	}

	// Close to flush remaining buffered bytes
	encoder.close()

	// Read all encoded bytes
	mut encoded := []u8{}
	mut read_buf := []u8{len: 4096}
	for {
		n := encoder.read(mut read_buf) or { break }
		encoded << read_buf[..n]
	}

	println('Original size: ${original_bytes.len} bytes')
	println('Encoded size:  ${encoded.len} bytes')

	// --- Decoding ---
	mut decoder := snappv.StreamDecoder{}

	// Write the encoded data (could also be done incrementally)
	decoder.write(encoded) or { panic('decode write error: ${err}') }
	decoder.close()

	// Read all decoded bytes
	mut decoded := []u8{}
	mut dec_read_buf := []u8{len: 4096}
	for {
		n := decoder.read(mut dec_read_buf) or { break }
		decoded << dec_read_buf[..n]
	}

	decoded_text := decoded.bytestr()

	println('Decoded text: "${decoded_text}"')
	println('Round-trip OK: ${original_text == decoded_text}')
}
```

## API

| Function | Description |
|---|---|
| `compress(data []u8) ![]u8` | Compresses data using Snappy raw block format |
| `decompress(data []u8) ![]u8` | Decompresses a Snappy raw block |
| `encode_stream(data []u8) ![]u8` | Compresses data using Snappy framing format |
| `decode_stream(data []u8) ![]u8` | Decompresses a Snappy framed stream |

## CRC32C Table Generation

The static CRC32C lookup table used for checksum computation in the framing format was generated by the Python script located at:

```
./scripts/generate_crc32c_table.py
```

If you need to regenerate the table (e.g., for verification), run:

```bash
python3 ./scripts/generate_crc32c_table.py > src/crc32c_table.v
```
